## Breakdown: Setiap Section

### 1️⃣ `.next/tw-classes/` — tailwind-styled-v4 Magic

**What**: Semua pre-generated CSS, state rules, dan metadata

```
_start.txt              13 bytes   Dev session start timestamp
_initial-scan.css       650KB      Tailwind safelist (3500 lines)
_tw-state-static.css    2KB        20 pre-generated state rules
_tw-build.log           1KB        Build log (empty on success)
_cycle.txt              13 bytes   Per-Turbopack-cycle timestamp
```

**Purpose**: These files exist ONLY at build time and dev time. They're NOT shipped to browser except `_tw-state-static.css`.

---

### 2️⃣ `.next/static/css/tw/` — Per-Route CSS (tailwind-styled-v4 Magic)

**What**: Route-specific CSS splitting

**Files**:
```
_global.css                           Shared by all routes
css-manifest.json                     Route → CSS mapping
route_root.css                        / route CSS
route_learn.css                       /learn route CSS
route_learn_mentor.css                /learn/mentor route CSS
route_learn_mentor_roadmap.css        /learn/mentor/roadmap CSS
... (40+ routes)
```

**Example manifest**:
```json
{
  "routes": {
    "__global": "_global.css",
    "/": "route_root.css",
    "/learn": "route_learn.css",
    "/learn/mentor": "route_learn_mentor.css"
  }
}
```

**Size comparison**:
```
_global.css          → ~20KB (shared by ALL routes)
route_root.css       → ~8KB  (ONLY for / route)
route_learn.css      → ~12KB (ONLY for /learn route)
route_learn_mentor.css → ~10KB (ONLY for /learn/mentor)
```

**Benefit**: Each route loads ONLY its CSS, not everything

## What's tailwind-styled-v4 Magic vs Next.js Magic?

### tailwind-styled-v4 Magic:
✅ `.next/tw-classes/` — _initial-scan.css, _tw-state-static.css  
✅ `.next/static/css/tw/` — Per-route CSS files + manifest  

### Next.js/Turbopack Magic:
✅ `.next/build/` — Turbopack compilation  
✅ `.next/dev/` — Dev server state  
✅ `.next/server/` — SSR compilation  
✅ `.next/static/chunks/` — Code splitting  
✅ Prerendering `.next/server/app/*.html`  

### Shared:
✅ `.next/cache/` — Build cache (both use it)  
✅ `.next/static/` — Asset delivery (CSS includes tw files)

## How It All Works Together

```
source code (src/)
    ↓ (tailwind-styled-v4 scanning)
_initial-scan.css + css-manifest.json
    ↓ (Tailwind PostCSS)
route_*.css files (per-route CSS)
    ↓ (Next.js bundling)
.next/static/chunks/ + .next/static/css/tw/
    ↓ (Browser download)
index.html + route.js + route.css
    ↓ (React hydrate)
Interactive app with pre-generated state CSS ✨
```
