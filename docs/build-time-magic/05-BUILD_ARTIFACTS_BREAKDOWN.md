```
examples/next-js-app/.next/tw-classes/
```

## File 1: `_start.txt`

**Size**: 13 bytes  
**Content**:
```
1783014112750
```

**Purpose**: Timestamp when dev server started

**Used by**: Turbopack loader to detect "is this same dev session or new restart?"

**Why**: 
- Turbopack caches aggressively
- On restart, cache becomes stale
- Loader compares _start.txt with last cycle's timestamp
- If different → clear cache

**Impact**: Prevents stale CSS from old dev sessions

## File 2: `_initial-scan.css`

**Size**: ~3500 lines, ~650KB (unminified)  
**Minified**: ~150KB  
**Loaded by browser**: NO (only used at build time)

**Sample content** (first 50 lines):

```css
/* tw-classes: initial scan — auto-generated by withTailwindStyled */
@layer utilities {
  .pointer-events-auto { pointer-events: auto; }
  .pointer-events-none { pointer-events: none; }
  .invisible { visibility: hidden; }
  .visible { visibility: visible; }
  .sr-only {
    position: absolute;
    width: 1px;
    height: 1px;
    padding: 0;
    margin: -1px;
    overflow: hidden;
    clip-path: inset(50%);
    white-space: nowrap;
    border-width: 0;
  }
  .absolute { position: absolute; }
  .fixed { position: fixed; }
  .relative { position: relative; }
  .static { position: static; }
  .sticky { position: sticky; }
  /* ... 3450+ more ... */
  .border-rose-300 { border-color: var(--color-rose-300); }
  .w-full { width: 100%; }
  .flex { display: flex; }
  .gap-4 { gap: calc(var(--spacing) * 4); }
  /* ... etc ... */
}
```

**What is this?**
- Tailwind's "safelist" — telling Tailwind "please generate CSS for these classes"
- Scanner extracted all Tailwind class strings from user source
- Passed to Tailwind PostCSS plugin
- Tailwind generates complete CSS for only these classes

**Example extraction**:

Input source:
```tsx
const Button = tw.button`
  inline-flex items-center px-4 py-2
  bg-blue-600 hover:bg-blue-700
  rounded-lg transition
`
```

Scanner extracts:
```
["inline-flex", "items-center", "px-4", "py-2", "bg-blue-600", "hover:bg-blue-700", "rounded-lg", "transition"]
```

_initial-scan.css includes CSS for all these classes.

**Why this approach?**
- Tailwind v4 doesn't have `content` option (removed from v4)
- Instead: pass class names via CSS file
- Tailwind PostCSS reads this file
- Generates only CSS it needs
- Result: tree-shaken output, no bloat

**What gets optimized away?**

Classes in _initial-scan.css NOT used by source:
- Tailwind removed these
- Example: if source never uses `.w-96`, it's not in final CSS

Classes in source that don't match Tailwind:
- Filtered out by false positive filter
- Example: `"header:topBar"` (sub-component key, not a Tailwind class)

## File 4: `css-manifest.json` (optional)

**Size**: ~1KB  
**Only generated if**: `routeCss: true` in next.config.ts  
**Loaded by browser**: NO (only server-side)  
**Used by**: TwCssInjector component

**Sample content**:

```json
{
  "/": "tw-route-abc123.css",
  "/learn": "tw-route-def456.css",
  "/learn/mentor": "tw-route-ghi789.css",
  "__global": "tw-global.css"
}
```

**How it's used**:

```tsx
// At request time (server-side)
const TwCssInjector = () => {
  const manifest = loadManifest()  // read css-manifest.json
  const pathname = usePathname()
  const cssFile = manifest[pathname]
  
  return (
    <style dangerouslySetInnerHTML={{ 
      __html: readCss(cssFile) 
    }} />
  )
}
// <head>
//   <style>{/* inline CSS for current route */}</style>
// </head>
```

**Benefit**: 
- No extra HTTP request for CSS
- Instant visual rendering
- Route-specific CSS only (smaller bundle)

## How This Gets to Browser

### Step 1: Build Time

```
source code (src/)
    ↓ (Rust scanner)
_initial-scan.css (3500 lines)
_tw-state-static.css (20 rules)
    ↓ (Tailwind PostCSS)
dist/index.css (final CSS)
```

### Step 2: Server Rendering

```
globals.css imports:
  ├─ @import "tailwindcss"
  ├─ @import ".next/tw-classes/_initial-scan.css"  (during build only)
  └─ @import ".next/tw-classes/_tw-state-static.css"
```

### Step 3: Delivery to Browser

```html
<!-- Option 1: Without TwCssInjector (simple) -->
<head>
  <link rel="stylesheet" href="/_next/static/css/dist/index.css" />
</head>

<!-- Option 2: With TwCssInjector (optimized) -->
<head>
  <style>
    /* tw-route-abc123.css content (inline) */
    .inline-flex { ... }
    .px-4 { ... }
    .tw-s-b35937[data-loading="true"] { ... }
  </style>
</head>
```

### Step 4: Runtime

```tsx
<Button loading>Process</Button>
// HTML: <button class="tw-..." data-loading="true">
// CSS matches: .tw-s-b35937[data-loading="true"]
// Styles: opacity: 60%; cursor: wait;
// Result: Visual feedback instantly! ✨
```

## Key Insight: The Trick

The magic happens because:

1. ✅ **Rust scanner is fast** (50ms vs 800ms JS)
2. ✅ **Pre-generation eliminates runtime overhead** (20 rules vs potential 1000+)
3. ✅ **Static analysis enables optimization** (route attribution, tree-shaking)
4. ✅ **Build-time work → runtime simplicity** (just set attributes, CSS does rest)

Result: Developer experience is simple, but the engine is doing sophisticated work! 🎯
