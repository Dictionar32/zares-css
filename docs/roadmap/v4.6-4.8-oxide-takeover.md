# v4.6–v4.8 Oxide Takeover (High-Speed Build Path)

Roadmap ini memfokuskan percepatan pipeline dengan pendekatan **Oxc-first, fallback-safe**.

## v4.6 — Oxide Engine Core (prototype)

- `tw parse <file>` menggunakan `scripts/v46/parse.mjs` (Oxc parser jika tersedia, fallback regex bila tidak).
- `tw transform <file> [out]` menggunakan `scripts/v46/transform.mjs` (Oxc transform bila tersedia).

## v4.7 — Compiler Optimization (prototype)

- `tw minify <file>` menggunakan `scripts/v47/minify.mjs` (Oxc minify bila tersedia).
- `tw shake <css-file>` untuk membersihkan rule CSS sentinel-unused.

## v4.8 — Parallel & Ecosystem (prototype)

- `tw lint [dir] [workers]` via `scripts/v48/lint-parallel.mjs`.
- `tw format <file> [--write]` via `scripts/v48/format.mjs`.
- `tw lsp` placeholder server command untuk integrasi LSP increment berikutnya.
- `tw benchmark` menghasilkan `docs/benchmark/toolchain-comparison.json`.

## Prinsip

1. Buildable cepat di repo saat ini.
2. Tidak memaksa native dependency agar CI tetap stabil.
3. Upgrade ke native/full Oxc dilakukan bertahap per package.
