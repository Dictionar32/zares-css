# v4.8–v5.0 Speed Styling Roadmap

Roadmap ini melanjutkan akselerasi `tw` dari parallel tooling menuju compile-time dan distributed build path.

Untuk visual kickoff lintas fase, lihat `docs/roadmap/visual-arsitektur-awal.md`.

## v4.8 (Parallel Oxide)

- `tw parse`, `tw transform`, `tw lint`, `tw format`, `tw benchmark`
- Fokus: parse paralel + feedback cepat untuk loop developer.

## v4.9 (Compile-time Optimization)

- `tw optimize <file> [--constant-folding] [--partial-eval]`
- `tw split [root] [outDir]`
- `tw critical <html> <css>`
- Fokus: precompute styling dan chunking CSS lebih agresif.

## v5.0 (Distributed Compilation)

- `tw cache <enable|status> [remote]`
- `tw cluster <init|build> [workers]`
- `tw adopt <feature> [project]`
- `tw metrics [port]`
- Fokus: cache remote + orchestration + incremental adoption.

## Catatan implementasi

- Semua command di atas disiapkan dalam mode prototype buildable.
- Integrasi production-grade (S3/Redis/worker orchestration sesungguhnya) dapat diiterasi bertahap tanpa breaking changes CLI.


## Status implementasi saat ini

- Engine plugin pipeline sudah menangani return value berbasis `undefined` check sehingga nilai valid yang falsy tetap diterapkan.
- Error handling engine sudah dinormalisasi melalui jalur tunggal untuk scan/build/watch dan hook `onError`.
- Opsi `tailwindConfigPath` kini dipakai untuk memuat konfigurasi Tailwind eksplisit pada flow build/watch.
- Fitur command v4.9/v5.0 lainnya masih berada pada tahap roadmap/prototype terpisah.
